# Security Policy & Governance Framework Project

## Project Overview
This project demonstrates the development of an **organizational information security governance framework** aligned with industry-recognized cybersecurity standards including:

- NIST Cybersecurity Framework (CSF)
- ISO 27001 Information Security Management System
- CIS Critical Security Controls

The objective of this project was to establish a **structured security governance model** that defines policies, responsibilities, and procedures for managing cybersecurity risk across the organization.

This project simulates the work typically performed by **Governance, Risk, and Compliance (GRC) Analysts** responsible for developing security policies, aligning organizations with compliance frameworks, and improving overall security maturity.

---

# Objectives
- Develop core **organizational security policies**
- Align policies with major **cybersecurity frameworks**
- Define **security governance roles and responsibilities**
- Create a **security awareness training framework**
- Improve **cybersecurity maturity and audit readiness**

---

# Organization Scenario
The organization modeled in this project is a **mid-sized technology services company** that manages:

- Customer personally identifiable information (PII)
- Internal financial and operational data
- Intellectual property
- Employee records

Because the company stores and processes sensitive information, a **formal information security governance framework** is required to standardize security practices and reduce organizational risk.

---

# Security Framework Alignment

This governance framework was mapped to multiple cybersecurity standards to ensure best practice adoption.

### NIST Cybersecurity Framework
Policies support the five core NIST functions:

- Identify
- Protect
- Detect
- Respond
- Recover

### ISO 27001
ISO 27001 provides a structured approach to building an **Information Security Management System (ISMS)** and managing security risks through governance, documentation, and continuous improvement.

### CIS Critical Security Controls
CIS Controls provide prioritized best practices designed to defend against common cyber threats.

---

# Security Policies Developed

The following foundational policies were created as part of this governance framework:

### Access Control Policy
Defines how users are granted, modified, and revoked access to company systems.

Key controls:
- Role Based Access Control (RBAC)
- Least privilege access
- Periodic access reviews

---

### Incident Response Policy
Defines procedures for identifying and responding to cybersecurity incidents.

Key components:
- Incident detection
- Incident reporting procedures
- Escalation paths
- Incident containment and recovery

---

### Data Protection Policy
Defines requirements for protecting sensitive information.

Key controls:
- Data classification
- Encryption requirements
- Secure data storage
- Secure data transmission

---

### Acceptable Use Policy
Defines appropriate usage of company technology resources.

Includes:
- Acceptable device usage
- Prohibited activities
- Network usage guidelines
- Monitoring expectations

---

### Password Policy
Defines authentication security requirements.

Controls include:
- Password complexity requirements
- Password rotation policies
- Multi-factor authentication (MFA)

---

### Security Awareness Policy
Defines employee cybersecurity training requirements.

Employees are trained on:

- Phishing awareness
- Password security
- Social engineering risks
- Incident reporting procedures

---

# Governance Structure

An organizational governance model was established to ensure security accountability.

| Role | Responsibility |
|-----|-----|
| Chief Information Security Officer (CISO) | Oversees cybersecurity strategy and policy enforcement |
| IT Security Team | Implements technical controls and monitors compliance |
| Human Resources | Enforces security training and employee compliance |
| Business Unit Leaders | Ensure departmental policy adherence |

---

# Security Awareness Program

A company-wide **security awareness training program** was introduced to reduce human-related cyber risk.

Training topics include:

- Phishing attack recognition
- Password best practices
- Data protection responsibilities
- Social engineering attacks
- Security incident reporting

Employees complete training:

- During onboarding
- Annually thereafter
- During periodic phishing simulation exercises
