# Third-Party Vendor Risk Assessment Project

## Overview
This project demonstrates a **Third-Party Vendor Risk Assessment** performed as part of a Governance, Risk, and Compliance (GRC) program. The objective of the project is to evaluate the cybersecurity posture of third-party vendors and determine the potential risks they pose to an organization's systems, data, and regulatory compliance.

Third-party vendors often have access to sensitive company data, internal systems, or financial information. As a result, organizations must perform structured vendor risk assessments to ensure vendors follow appropriate security practices and industry compliance standards.

This project simulates a real-world **vendor risk management process** used by security and compliance teams.

---

## Objectives
- Identify third-party vendors and their access to company systems and data  
- Evaluate vendor cybersecurity maturity and compliance posture  
- Review vendor security documentation such as SOC 2 reports and security policies  
- Assign risk scores based on likelihood and business impact  
- Identify security gaps in vendor controls  
- Provide mitigation recommendations to reduce vendor risk  

---

## Project Components

### 1. Vendor Inventory
A vendor inventory was created to identify all third-party vendors and document their:

- Services provided
- Data access level
- System access permissions
- Business ownership
- Vendor criticality level

This step ensures the organization understands which vendors interact with sensitive systems or data.

---

### 2. Vendor Criticality Classification
Vendors were categorized based on the potential business impact of a vendor security incident.

| Risk Level | Description |
|---|---|
| Critical | Access to financial systems or payment infrastructure |
| High | Access to sensitive customer or employee data |
| Medium | Limited system or data access |
| Low | No system access |

This classification helps prioritize vendor risk reviews.

---

### 3. Security Questionnaire
A vendor security questionnaire was used to evaluate vendor cybersecurity practices.  
The questionnaire assessed controls such as:

- Information security policies
- Encryption practices
- Multi-factor authentication
- Incident response procedures
- Security awareness training
- Vulnerability management
- Penetration testing

The responses help determine vendor security maturity.

---

### 4. Security Documentation Review
Vendor documentation was reviewed to validate security practices and compliance claims.

Documents reviewed included:

- SOC 2 Type II reports
- ISO 27001 certifications
- Security policies
- Incident response plans
- Penetration testing summaries

This step verifies that vendors maintain appropriate security frameworks.

---

### 5. Vendor Risk Scoring
A risk scoring model was used to quantify vendor risk.

Risk Score = Likelihood × Impact

| Score Range | Risk Level |
|---|---|
| 1 – 4 | Low |
| 5 – 9 | Medium |
| 10 – 16 | High |
| 17 – 25 | Critical |

This approach allows organizations to prioritize vendors that present the highest potential risk.

---

### 6. Security Gap Identification
The assessment identified several vendor security gaps, including:

- Lack of compliance certifications
- Weak encryption controls
- Incomplete security documentation

Identifying these gaps allows organizations to implement appropriate remediation strategies.

---

### 7. Risk Mitigation Recommendations
Based on the findings, mitigation strategies were recommended, such as:

- Requiring vendors to obtain SOC 2 certification
- Strengthening encryption practices
- Requesting penetration testing reports
- Increasing vendor monitoring

These recommendations help reduce third-party cybersecurity risk.

---

### 8. Continuous Vendor Monitoring
Vendor risk management is an ongoing process. Organizations should implement continuous monitoring through:

- Annual vendor reassessments
- Updated security questionnaires
- Compliance documentation reviews
- Vendor breach monitoring

Continuous monitoring ensures vendor security practices remain aligned with organizational risk tolerance.
