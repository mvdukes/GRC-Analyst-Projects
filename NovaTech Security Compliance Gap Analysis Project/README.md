# NovaTech Security Compliance Gap Analysis Project

**Company:** NovaTech Logistics, Inc.

---

## Project Overview

This project demonstrates a simulated **enterprise cybersecurity risk assessment** conducted for NovaTech Logistics, Inc., a mid-sized logistics technology company. The objective of this assessment was to identify cybersecurity risks, evaluate existing security controls, and recommend remediation strategies aligned with recognized industry frameworks.

The assessment was structured using guidance from:

- NIST Cybersecurity Framework (NIST CSF)  
- ISO 27001 Information Security Management concepts  
- NIST SP 800-53 security control mapping  

This project showcases core **Governance, Risk, and Compliance (GRC)** skills including risk identification, control evaluation, risk scoring, remediation planning, and framework alignment.

---

## Objectives

The primary objectives of this project were to:

- Identify critical organizational assets
- Evaluate existing cybersecurity controls
- Perform risk identification and analysis
- Score risks based on likelihood and impact
- Document findings within a structured risk register
- Develop remediation recommendations
- Map security improvements to NIST 800-53 controls

## Assessment Methodology

The cybersecurity risk assessment followed a structured methodology based on recognized frameworks.

### Framework Alignment

| Framework | Purpose |
|--------|--------|
| NIST CSF | Risk management structure |
| ISO 27001 | Information security governance |
| NIST SP 800-53 | Control implementation guidance |

### Risk Scoring Methodology

Risks were evaluated using a **Likelihood × Impact** scoring model.

| Score Range | Risk Level |
|-------------|------------|
| 1–5 | Low |
| 6–10 | Moderate |
| 11–15 | High |
| 16–25 | Critical |

## Project Components

This project includes several deliverables commonly produced during enterprise risk assessments.

### 1. Asset Inventory
Identifies critical organizational assets that support core operations.

Examples:
- Employee laptops
- Cloud logistics management system
- Microsoft 365 email
- Shipment tracking database
- Vendor access accounts

---

### 2. Control Gap Analysis
Evaluates existing cybersecurity controls and identifies missing or incomplete protections.

Example gaps identified:
- Multi-factor authentication not enabled
- Limited security monitoring
- No formal incident response plan
- Vendor access not reviewed regularly

---

### 3. Risk Register
Documents identified risks, including threats, vulnerabilities, likelihood, impact, and remediation actions.

Example risks:

| Risk | Description |
|-----|-------------|
| Phishing attacks | Credential compromise through email phishing |
| Unauthorized vendor access | Vendors retaining unnecessary system permissions |
| Cloud data exposure | Improper file sharing permissions |
| Incident response delays | Lack of formal response procedures |

---

### 4. Remediation Plan
Provides recommendations to reduce organizational risk exposure.

Key recommendations include:

- Implement multi-factor authentication (MFA)
- Deploy centralized monitoring and logging
- Conduct quarterly vendor access reviews
- Implement least privilege access controls
- Develop an incident response plan
- Implement employee security awareness training

---

### 5. NIST 800-53 Control Mapping

Remediation actions were mapped to relevant NIST controls to align with industry best practices.

| Improvement | NIST Control |
|-------------|--------------|
| Multi-factor authentication | IA-2 Identification and Authentication |
| Vendor access reviews | AC-2 Account Management |
| Least privilege policies | AC-6 Least Privilege |
| Security awareness training | AT-2 Security Awareness |
| Incident response planning | IR-8 Incident Response Plan |
